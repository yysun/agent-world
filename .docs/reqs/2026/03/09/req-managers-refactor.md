# REQ: Consolidate and Refactor managers.ts

**Date:** 2026-03-09  
**Status:** Draft  

---

## Problem Statement

`core/managers.ts` has grown into a ~2960-line, 42-export god module mixing six distinct domains:
world CRUD, agent CRUD, chat CRUD + restore, message editing, message ID migration, and queue dispatch.
The size and mixing of concerns makes the file difficult to navigate, test, and extend safely.

---

## Requirements

### R1 — Extract queue management into a dedicated module

The queue dispatch domain (`core/queue-manager.ts` or equivalent) shall contain:
- All module-level queue state (paused sets, in-flight sets, dispatch state map, retry handles, listener maps).
- All private queue helpers: `triggerPendingQueueResume`, `attachQueueAdvanceListener`, `detachQueueAdvanceListener`, `scheduleQueueNoResponseFallback`, `handleQueueDispatchFailure`, `runQueueResponderPreflight`, `refreshQueueRespondersFromStorage`, `collectQueueAgentStatus`, `hasQueueResponderAvailability`, `resolveQueueMainAgentId`, `normalizeQueueMentionToken`, `getQueueStorageOrThrow`, `getQueueKey`, `getQueueMessageKey`, `clearQueueRetryStatusEmitter`, `startQueueRetryStatusEmitter`, `clearQueueResponderRefreshAttempt`, `isUserQueueSender`.
- All public queue exports: `addToQueue`, `recoverQueueSendingMessages`, `enqueueAndProcessUserMessage`, `getQueueMessages`, `removeFromQueue`, `pauseChatQueue`, `resumeChatQueue`, `stopChatQueue`, `clearChatQueue`, `retryQueueMessage`.
- The private queue types: `QueueDispatchState`, `QueueRetryStatusHandle`, `QueueAgentStatusSnapshot`, `QueueStorageOperations`.

The extracted module shares `storageWrappers` / `ensureInitialization` via an injected getter or re-exported accessor, keeping initialization in `managers.ts`.

### R2 — Extract message editing into a dedicated module

A `core/message-edit-manager.ts` (or equivalent) shall contain:
- `removeMessagesFrom`, `editUserMessage`, `migrateMessageIds`.
- `logEditError`, `getEditErrors` — **but** these must be refactored to route through the storage abstraction layer (StorageAPI) instead of raw `fs.readFileSync`/`fs.writeFileSync`. This satisfies the "always use in-memory storage in tests" rule.
- The private helpers: `resetAutoGeneratedChatTitleForEditResubmission`, `rollbackAutoGeneratedChatTitleResetAfterFailedResubmission`, `syncRuntimeAgentMemoryFromStorage`, `extractGeneratedChatTitleFromSystemPayload`.
- The private type: `EditResubmissionTitleResetResult`.

### R3 — De-duplicate the `restoreChat` activation sequence

The two nearly-identical branches in `restoreChat` (already-current path and switch path) shall be consolidated into a single private `activateChatResources(world, chatId)` helper covering: memory sync → skill approval reconstruction → HITL replay → pending-last-message resume → queue un-pause and resume.

### R4 — Replace repeated dynamic imports with static top-level imports

`subscription.js` and `events/index.js` are dynamically imported (10+ times each) via `await import(...)`. These shall be converted to static top-level `import` statements to eliminate the per-call overhead and fix circular-dependency ambiguity.

> **Constraint:** Circular import cycles must be verified before converting any dynamic import to static. If a static import would introduce a cycle, keep the dynamic import and document why.

### R5 — `managers.ts` remains the public entry point

`core/managers.ts` shall continue to re-export (or forward) all currently exported symbols so no callers (`server/api.ts`, `cli/`, `electron/`, `web/`, tests) require any import path changes.

### R6 — All existing tests must continue to pass

No behavior changes are introduced. This is a structural refactor only. All existing unit and integration tests must pass after the changes.

### R7 — New tests for the refactored boundaries

Each new module (`queue-manager`, `message-edit-manager`) shall have at least one targeted unit test verifying its public API boundary.

---

## Out of Scope

- Changing any public function signatures or return types.
- Changing the `core/index.ts` export surface visible to npm consumers.
- Refactoring `branchChatFromMessage` internals beyond the extract described above.
- Any feature additions or behavior changes.
- Trimming the header changelog (low value vs. risk; deferred).

---

## Success Criteria

1. `managers.ts` is reduced to ≤ 1200 lines.
2. `queue-manager.ts` and `message-edit-manager.ts` exist as cohesive, focused modules.
3. `logEditError` / `getEditErrors` no longer use raw `fs` calls.
4. `restoreChat` has a single activation helper (no duplicated branch logic).
5. All `npm test` and `npm run integration` checks pass.
